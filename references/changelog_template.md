# 产品更新日志

> 只能输出正文，不要添加前言、统计信息、生成时间、仓库列表或工具说明。
> 只能使用 `✨ 新功能`、`🔄 功能变更`、`🔧 技术改造`、`🐛 Bug 修复` 四类标题。
> 禁止输出 `### 📝 其他更新`、`feat(`、`fix(`、`refactor(`、DTO、SQL、类名、接口路径。
> 单日生成时先产出一个日期块；最终汇总时文件顶部只保留一次 `# 产品更新日志`。

## [YYYY-MM-DD]
（仅当此日期有变更内容时才输出）

### ✨ 新功能

- （有则填写，无则删除此分类）

### 🔄 功能变更

- （有则填写，无则删除此分类）

### 🔧 技术改造

- （有则填写，无则删除此分类）

### 🐛 Bug 修复

- （有则填写，无则删除此分类）

## 合格示例

```markdown
# 产品更新日志

## 2026-03-11

### ✨ 新功能

- 列表页新增状态筛选能力，运营人员可按条件快速定位目标记录

### 🐛 Bug 修复

- 修复统计页中的时间排序异常，最近数据展示更准确
```

## 常见错误改写

- `feat(resource): 新增申请功能` -> `资源模块新增申请入口，用户可提交申请`
- `SQL 优化` -> `相关查询效率提升，列表加载更稳定`
- `DTO 字段调整` -> `页面展示字段和返回信息已同步调整`
- `/orders` -> `订单页` 或 `订单模块`
- `ExampleController` -> `后台相关处理能力` 或具体业务动作
